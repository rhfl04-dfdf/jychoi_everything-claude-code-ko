# PHP 패턴

> 이 파일은 [common/patterns.md](../common/patterns.md)를 확장하여 PHP 특화 내용을 다룹니다.

## 얇은 컨트롤러, 명시적 서비스

- 컨트롤러는 전송에 집중: 인증, 검증, 직렬화, 상태 코드
- 비즈니스 규칙은 HTTP 부트스트래핑 없이 테스트하기 쉬운 애플리케이션/도메인 서비스로 이동

## DTO와 값 객체

- 요청, 명령, 외부 API 페이로드에 형태가 복잡한 연관 배열 대신 DTO 사용
- 금액, 식별자, 날짜 범위 등 제한된 개념에는 값 객체 사용

## 의존성 주입

- 프레임워크 전역 변수가 아닌 인터페이스 또는 좁은 서비스 계약에 의존
- 서비스가 서비스 로케이터 조회 없이 테스트 가능하도록 생성자를 통해 협력자 전달

## 경계

- 모델 레이어가 영속성 이상을 담당할 때 ORM 모델을 도메인 결정으로부터 분리
- 나머지 코드베이스가 서드파티 계약이 아닌 자체 계약에 의존하도록 서드파티 SDK를 작은 어댑터로 래핑
