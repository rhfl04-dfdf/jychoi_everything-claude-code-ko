# 투자자 자료

일관성 있고, 신뢰할 수 있으며, 방어하기 쉬운 투자자용 자료를 만듭니다.

## 활성화 시점

- 피치 덱 생성 또는 수정
- 투자자 메모 또는 원페이저 작성
- 재무 모델, 마일스톤 계획, 자금 사용 계획표 작성
- 액셀러레이터 또는 인큐베이터 지원 질문 답변
- 하나의 진실 소스를 중심으로 여러 펀드레이징 문서 정렬

## 황금 법칙

모든 투자자 자료는 서로 일치해야 합니다.

작성 전에 단일 진실 소스를 만들거나 확인하세요:
- 트랙션 지표
- 가격 및 수익 가정
- 투자 유치 규모 및 수단
- 자금 사용처
- 팀 약력 및 직함
- 마일스톤 및 타임라인

상충되는 숫자가 나타나면, 초안 작성 전에 멈추고 해결하세요.

## 핵심 워크플로우

1. 표준 사실 목록 작성
2. 누락된 가정 확인
3. 자료 유형 선택
4. 명시적인 논리로 자료 초안 작성
5. 모든 숫자를 진실 소스와 대조 확인

## 자료별 가이드

### 피치 덱
권장 흐름:
1. 회사 + 진입점
2. 문제
3. 솔루션
4. 제품 / 데모
5. 시장
6. 비즈니스 모델
7. 트랙션
8. 팀
9. 경쟁 / 차별화
10. 요청
11. 자금 사용처 / 마일스톤
12. 부록

웹 네이티브 덱을 원한다면 이 skill을 `frontend-slides`와 함께 사용하세요.

### 원페이저 / 메모
- 회사가 무엇을 하는지 깔끔한 한 문장으로 설명하세요
- 지금이 적기인 이유를 보여주세요
- 트랙션과 증거 포인트를 초반에 포함하세요
- 요청을 구체적으로 하세요
- 주장을 쉽게 검증할 수 있게 하세요

### 재무 모델
포함 내용:
- 명시적인 가정
- 유용할 경우 최악/기본/최선의 시나리오
- 깔끔한 레이어별 수익 논리
- 마일스톤 연계 지출
- 의사결정이 가정에 달려 있는 경우 민감도 분석

### 액셀러레이터 지원서
- 물어본 질문에 정확히 답변하세요
- 트랙션, 인사이트, 팀 강점을 우선시하세요
- 과장을 피하세요
- 내부 지표를 덱 및 모델과 일관성 있게 유지하세요

## 피해야 할 위험 신호

- 검증 불가능한 주장
- 가정 없는 모호한 시장 규모 산정
- 일관성 없는 팀 역할 또는 직함
- 깔끔하게 합산되지 않는 수익 계산
- 가정이 불안정한 곳에서의 부풀려진 확실성

## 품질 기준

전달 전:
- 모든 숫자가 현재 진실 소스와 일치함
- 자금 사용처와 수익 레이어가 올바르게 합산됨
- 가정이 묻히지 않고 보임
- 과장 언어 없이 이야기가 명확함
- 최종 자료가 파트너 미팅에서 방어 가능함
