# Agentic Engineering

AI 에이전트가 대부분의 구현 작업을 수행하고 인간이 품질 및 위험 통제를 담당하는 엔지니어링 워크플로우에 이 스킬을 사용하세요.

## 운영 원칙

1. 실행 전에 완료 기준을 정의하세요.
2. 작업을 에이전트 단위로 분해하세요.
3. 작업 복잡도에 따라 모델 티어를 라우팅하세요.
4. eval과 회귀 검사로 측정하세요.

## Eval 우선 루프

1. 기능 eval과 회귀 eval을 정의하세요.
2. 베이스라인을 실행하고 실패 패턴을 캡처하세요.
3. 구현을 실행하세요.
4. eval을 재실행하고 델타를 비교하세요.

## 작업 분해

15분 단위 규칙을 적용하세요:
- 각 단위는 독립적으로 검증 가능해야 합니다
- 각 단위는 단일 주요 위험을 가져야 합니다
- 각 단위는 명확한 완료 조건을 드러내야 합니다

## 모델 라우팅

- Haiku: 분류, 보일러플레이트 변환, 좁은 범위의 편집
- Sonnet: 구현 및 리팩토링
- Opus: 아키텍처, 근본 원인 분석, 멀티 파일 불변성

## 세션 전략

- 긴밀하게 연결된 단위에 대해서는 세션을 계속 유지하세요.
- 주요 단계 전환 후에는 새 세션을 시작하세요.
- 활성 디버깅 중이 아닌 마일스톤 완료 후에 컴팩트(compact)하세요.

## AI 생성 코드 리뷰 포커스

다음을 우선시하세요:
- 불변성 및 엣지 케이스
- 에러 경계
- 보안 및 인증 가정
- 숨겨진 결합 및 롤아웃 위험

자동화된 형식/린트가 이미 스타일을 강제하는 경우 스타일만의 불일치에 리뷰 사이클을 낭비하지 마세요.

## 비용 규율

작업별로 다음을 추적하세요:
- 모델
- 토큰 추정치
- 재시도 횟수
- 실제 소요 시간
- 성공/실패

하위 티어가 명확한 추론 격차로 실패할 때만 모델 티어를 상향 조정하세요.
