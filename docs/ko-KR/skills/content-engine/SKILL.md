# Content Engine

하나의 아이디어를 모든 곳에 똑같이 게시하는 대신, 강력한 플랫폼 네이티브 콘텐츠로 전환하세요.

## When to Activate

- X 포스트 또는 스레드 작성
- LinkedIn 포스트 또는 출시 업데이트 초안 작성
- 숏폼 비디오 또는 YouTube 설명 영상 스크립트 작성
- 아티클, 팟캐스트, 데모 또는 문서를 소셜 콘텐츠로 repurposing
- 출시, 마일스톤 또는 테마를 중심으로 가벼운 콘텐츠 계획 수립

## First Questions

다음을 명확히 하세요:
- source asset: 무엇을 기반으로 변환하나요?
- audience: 빌더, 투자자, 고객, 운영자 또는 일반 대중
- platform: X, LinkedIn, TikTok, YouTube, 뉴스레터 또는 멀티 플랫폼
- goal: 인지도, 전환, 채용, 권위, 출시 지원 또는 참여

## Core Rules

1. 플랫폼에 맞게 조정하세요. 동일한 카피를 여러 곳에 중복 게시하지 마세요.
2. 요약보다 hook이 더 중요합니다.
3. 모든 포스트는 하나의 명확한 아이디어를 담아야 합니다.
4. 슬로건보다는 구체적인 내용을 사용하세요.
5. 요청(CTA)은 작고 명확하게 유지하세요.

## Platform Guidance

### X
- 빠른 도입부
- 포스트 또는 스레드의 트윗당 하나의 아이디어
- 필요하지 않은 경우 본문에 링크를 넣지 마세요
- 해시태그 스팸을 피하세요

### LinkedIn
- 강력한 첫 줄
- 짧은 단락
- 교훈, 결과 및 시사점을 중심으로 더 명시적인 프레이밍

### TikTok / 숏폼 비디오
- 처음 3초 이내에 시선을 사로잡아야 합니다
- 내레이션뿐만 아니라 비주얼을 중심으로 스크립트 작성
- 하나의 데모, 하나의 주장, 하나의 CTA

### YouTube
- 결과를 초반에 보여주세요
- 챕터별 구조화
- 20~30초마다 시각적 변화 제공

### 뉴스레터
- 관련 없는 항목의 묶음이 아닌 하나의 명확한 관점을 전달하세요
- 섹션 제목을 훑어보기 쉽게 만드세요
- 도입부 단락이 실질적인 역할을 하도록 하세요

## Repurposing Flow

기본 단계:
1. anchor asset: 아티클, 비디오, 데모, 메모 또는 출시 문서
2. 3~7개의 핵심(atomic) 아이디어 추출
3. 플랫폼 네이티브 변형 작성
4. 결과물 간의 중복 제거
5. 플랫폼 의도에 맞게 CTA 정렬

## Deliverables

캠페인을 요청받으면 다음을 반환하세요:
- 핵심 앵글
- 플랫폼별 초안
- 선택적 게시 순서
- 선택적 CTA 변형
- 게시 전 필요한 누락된 입력 사항

## Quality Gate

전달하기 전:
- 각 초안이 해당 플랫폼에 네이티브하게 읽히는지 확인
- hook이 강력하고 구체적인지 확인
- 일반적인 과대광고 용어 지양
- 요청되지 않은 경우 플랫폼 간에 카피가 중복되지 않도록 함
- CTA가 콘텐츠 및 audience와 일치하는지 확인
