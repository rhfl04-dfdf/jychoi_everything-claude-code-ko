# 아티클 작성

일반적인 AI 출력물이 아닌, 실제 사람이나 브랜드처럼 들리는 장문 콘텐츠를 작성합니다.

## 활성화 시점

- 블로그 포스트, 에세이, 런치 포스트, 가이드, 튜토리얼, 뉴스레터 초안 작성
- 메모, 기록, 리서치를 완성도 높은 아티클로 전환
- 예시에서 기존 창업자, 운영자, 브랜드 보이스 매칭
- 이미 작성된 장문 카피의 구조, 흐름, 근거 보강

## 핵심 규칙

1. 구체적인 것으로 시작하세요: 예시, 출력물, 일화, 숫자, 스크린샷 설명, 코드 블록.
2. 예시 다음에 설명하세요, 이전이 아니라.
3. 늘어진 문장보다 짧고 직접적인 문장을 선호하세요.
4. 출처가 있고 사용 가능한 경우 구체적인 수치를 사용하세요.
5. 전기적 사실, 회사 지표, 고객 증거를 절대 만들어내지 마세요.

## 보이스 캡처 워크플로우

사용자가 특정 보이스를 원하는 경우 다음 중 하나 이상을 수집하세요:
- 발행된 아티클
- 뉴스레터
- X / LinkedIn 포스트
- 문서 또는 메모
- 짧은 스타일 가이드

그런 다음 추출하세요:
- 문장 길이와 리듬
- 보이스가 격식체, 대화체, 예리한지 여부
- 괄호, 목록, 단편, 질문 같은 선호하는 수사 기법
- 유머, 의견, 역발상 프레이밍에 대한 수용도
- 헤더, 불릿, 코드 블록, 인용 블록 같은 포매팅 습관

보이스 참고 자료가 없으면 직접적이고 운영자 스타일의 보이스를 기본으로 사용하세요: 구체적이고, 실용적이며, 과장 없음.

## 금지 패턴

다음은 삭제하고 다시 쓰세요:
- "오늘날 빠르게 변화하는 환경에서"와 같은 일반적인 도입부
- "더욱이", "그뿐만 아니라" 같은 채움 전환어
- "게임 체인저", "최첨단", "혁신적" 같은 과장 문구
- 근거 없는 모호한 주장
- 제공된 맥락으로 뒷받침되지 않는 전기적 또는 신뢰성 주장

## 작성 프로세스

1. 독자와 목적을 명확히 하세요.
2. 섹션당 하나의 목적을 가진 뼈대 개요를 만드세요.
3. 각 섹션을 근거, 예시, 또는 장면으로 시작하세요.
4. 다음 문장이 자리를 차지할 가치가 있을 때만 확장하세요.
5. 템플릿처럼 들리거나 자화자찬하는 내용은 모두 제거하세요.

## 구조 가이드

### 기술 가이드
- 독자가 얻는 것으로 시작
- 모든 주요 섹션에 코드 또는 터미널 예시 사용
- 부드러운 요약이 아닌 구체적인 핵심 사항으로 마무리

### 에세이 / 의견 글
- 긴장감, 모순, 또는 예리한 관찰로 시작
- 섹션당 하나의 주장 흐름 유지
- 의견을 뒷받침하는 예시 사용

### 뉴스레터
- 첫 화면을 강력하게 유지
- 인사이트와 업데이트 혼합, 일기 채움 없음
- 명확한 섹션 레이블과 쉽게 훑어볼 수 있는 구조 사용

## 품질 게이트

전달 전:
- 제공된 소스에 대한 사실적 주장 검증
- 채움 표현과 기업 언어 제거
- 보이스가 제공된 예시와 일치하는지 확인
- 모든 섹션이 새로운 정보를 추가하는지 확인
- 의도된 플랫폼에 맞는 포매팅 확인
