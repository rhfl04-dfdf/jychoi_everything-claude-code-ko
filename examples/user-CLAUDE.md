# 사용자 수준 CLAUDE.md 예제

이것은 사용자 수준의 CLAUDE.md 예제 파일입니다. `~/.claude/CLAUDE.md`에 배치하세요.

사용자 수준 설정은 모든 프로젝트에 전역으로 적용됩니다. 다음 용도로 사용하세요:
- 개인 코딩 선호도
- 항상 적용하고 싶은 보편적 규칙
- 모듈형 규칙 파일 링크

---

## 핵심 철학

당신은 Claude Code입니다. 복잡한 작업에는 특화된 agent와 skill을 사용합니다.

**핵심 원칙:**
1. **Agent 우선**: 복잡한 작업은 전문 agent에게 위임
2. **병렬 실행**: 가능한 경우 여러 agent와 Task 도구 사용
3. **실행 전 계획**: 복잡한 작업에는 Plan Mode 사용
4. **테스트 주도**: 구현 전에 테스트 작성
5. **보안 우선**: 보안에 절대 타협하지 않음

---

세부 가이드라인은 `~/.claude/rules/`에 있습니다:

| performance.md | 모델 선택, 컨텍스트 관리 |
| hooks.md | Hook 시스템 |

## 개인 설정

### 개인 정보 보호
- 항상 로그 검열; 시크릿(API 키/토큰/비밀번호/JWT) 붙여넣기 금지
- 공유 전 출력 검토 - 민감한 데이터 제거

### 코드 스타일
- 코드, 주석, 문서에 이모지 사용 금지
- 불변성 선호 - 객체나 배열 변경 금지
- 큰 파일 몇 개보다 작은 파일 여러 개
- 일반적으로 200~400줄, 파일당 최대 800줄

### Git
- 컨벤셔널 커밋: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- 커밋 전 항상 로컬에서 테스트
- 작고 집중된 커밋

### 테스트
- TDD: 테스트를 먼저 작성
- 최소 80% 커버리지
- 중요 흐름에 대한 단위 + 통합 + E2E 테스트

### 지식 관리
- 개인 디버깅 노트, 선호도, 임시 컨텍스트 → 자동 메모리
- 팀/프로젝트 지식(아키텍처 결정, API 변경, 구현 런북) → 프로젝트의 기존 문서 구조 따름
- 현재 작업에서 이미 관련 문서, 주석, 예시가 생성되는 경우 동일한 지식을 다른 곳에 중복하지 않음
- 명확한 프로젝트 문서 위치가 없는 경우, 새로운 최상위 문서를 생성하기 전에 먼저 질문

Zed를 주 에디터로 사용합니다:
- 파일 추적을 위한 Agent 패널
- 명령 팔레트에 CMD+Shift+R
- Vim 모드 활성화

## 성공 기준

다음 조건을 만족할 때 성공입니다:
- 모든 테스트 통과 (80%+ 커버리지)
- 보안 취약점 없음
- 코드가 읽기 쉽고 유지 관리 가능
- 사용자 요구사항 충족
