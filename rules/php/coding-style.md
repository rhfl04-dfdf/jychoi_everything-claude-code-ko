# PHP 코딩 스타일

> 이 파일은 [common/coding-style.md](../common/coding-style.md)를 확장하여 PHP 특화 내용을 다룹니다.

## 표준

- **PSR-12** 포맷팅 및 네이밍 컨벤션 준수
- 애플리케이션 코드에서 `declare(strict_types=1);` 선호
- 새 코드가 허용하는 모든 곳에 스칼라 타입 힌트, 반환 타입, 타입이 지정된 프로퍼티 사용

## 불변성

- 서비스 경계를 넘는 데이터에는 불변 DTO와 값 객체 선호
- 가능한 경우 요청/응답 페이로드에 `readonly` 프로퍼티 또는 불변 생성자 사용
- 단순 맵에는 배열 사용; 비즈니스 핵심 구조는 명시적 클래스로 승격

## 포맷팅

- 포맷팅에는 **PHP-CS-Fixer** 또는 **Laravel Pint** 사용
- 정적 분석에는 **PHPStan** 또는 **Psalm** 사용
- 동일한 명령이 로컬과 CI에서 실행되도록 Composer 스크립트를 체크인 유지

## 에러 처리

- 예외적인 상태에는 예외를 던짐; 새 코드에서 `false`/`null`을 숨겨진 에러 채널로 반환하지 않기
- 프레임워크/요청 입력을 도메인 로직에 도달하기 전에 검증된 DTO로 변환
