# TDD 명령어

엄격한 테스트 주도 개발 방식으로 다음을 구현합니다: $ARGUMENTS

## TDD 사이클 (필수)

```
RED → GREEN → REFACTOR → REPEAT
```

1. **RED**: 실패하는 테스트를 먼저 작성
2. **GREEN**: 테스트를 통과시키는 최소한의 코드 작성
3. **REFACTOR**: 테스트를 유지하면서 코드 개선
4. **REPEAT**: 기능 완성까지 반복

## 작업 내용

### 단계 1: 인터페이스 정의 (SCAFFOLD)
- 입력/출력에 대한 TypeScript 인터페이스 정의
- `throw new Error('Not implemented')`로 함수 시그니처 생성

### 단계 2: 실패하는 테스트 작성 (RED)
- 인터페이스를 실행하는 테스트 작성
- 정상 경로, 엣지 케이스, 오류 조건 포함
- 테스트 실행 - 실패하는지 확인

### 단계 3: 최소 코드 구현 (GREEN)
- 테스트를 통과시키기 위한 최소한의 코드 작성
- 성급한 최적화 금지
- 테스트 실행 - 통과하는지 확인

### 단계 4: 리팩토링 (IMPROVE)
- 상수 추출, 네이밍 개선
- 중복 제거
- 테스트 실행 - 여전히 통과하는지 확인

### 단계 5: 커버리지 확인
- 목표: 최소 80%
- 핵심 비즈니스 로직은 100%
- 필요 시 테스트 추가

## 커버리지 요구사항

| 코드 유형 | 최소 기준 |
|-----------|-----------|
| 일반 코드 | 80% |
| 금융 계산 | 100% |
| 인증 로직 | 100% |
| 보안 관련 코드 | 100% |

## 포함할 테스트 유형

- **단위 테스트**: 개별 함수
- **엣지 케이스**: 빈 값, null, 최대값, 경계값
- **오류 조건**: 잘못된 입력, 네트워크 장애
- **통합 테스트**: API 엔드포인트, 데이터베이스 작업

---

**필수**: 테스트는 반드시 구현 전에 작성해야 합니다. RED 단계를 절대 건너뛰지 마세요.
