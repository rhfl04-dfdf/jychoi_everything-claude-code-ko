# Verify 명령어

구현을 검증하기 위한 검증 루프를 실행합니다: $ARGUMENTS

## 작업 내용

포괄적인 검증을 실행합니다:

1. **타입 검사**: `npx tsc --noEmit`
2. **린트**: `npm run lint`
3. **단위 테스트**: `npm test`
4. **통합 테스트**: `npm run test:integration` (사용 가능한 경우)
5. **빌드**: `npm run build`
6. **커버리지 확인**: 80% 이상 커버리지 검증

## 검증 체크리스트

### 코드 품질
- [ ] TypeScript 오류 없음
- [ ] 린트 경고 없음
- [ ] console.log 구문 없음
- [ ] 함수 50줄 미만
- [ ] 파일 800줄 미만

### 테스트
- [ ] 모든 테스트 통과
- [ ] 커버리지 >= 80%
- [ ] 엣지 케이스 포함
- [ ] 오류 조건 테스트 완료

### 보안
- [ ] 하드코딩된 비밀 정보 없음
- [ ] 입력 유효성 검사 존재
- [ ] SQL injection 위험 없음
- [ ] XSS 취약점 없음

### 빌드
- [ ] 빌드 성공
- [ ] 경고 없음
- [ ] 번들 크기 적절

## 검증 보고서

### 요약
- 상태: ✅ PASS / ❌ FAIL
- 점수: X/Y 검사 통과

### 상세
| 검사 | 상태 | 비고 |
|------|------|------|
| TypeScript | ✅/❌ | [상세] |
| 린트 | ✅/❌ | [상세] |
| 테스트 | ✅/❌ | [상세] |
| 커버리지 | ✅/❌ | XX% (목표: 80%) |
| 빌드 | ✅/❌ | [상세] |

### 조치 사항
[FAIL인 경우, 수정이 필요한 항목 목록]
