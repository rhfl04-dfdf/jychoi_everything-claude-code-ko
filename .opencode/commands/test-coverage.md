# Test Coverage 명령어

테스트 커버리지를 분석하고 갭을 식별합니다: $ARGUMENTS

## 작업 내용

1. **커버리지 보고서 실행**: `npm test -- --coverage`
2. **결과 분석** - 낮은 커버리지 영역 식별
3. **갭 우선순위 지정** - 중요 코드 우선
4. **누락된 테스트 생성** - 미커버 코드 대상

## 커버리지 목표

| 코드 유형 | 목표 |

| 일반 코드 | 80% |

| 인증/보안 | 100% |
| 유틸리티 | 90% |

## 커버리지 개선 계획

1. **긴급** (즉시 추가)
   - [ ] file1.ts - 인증 로직
   - [ ] file2.ts - 결제 처리

2. **높음** (이번 스프린트에 추가)
   - [ ] file3.ts - 핵심 비즈니스 로직

3. **중간** (파일 수정 시 추가)
   - [ ] file4.ts - 유틸리티
