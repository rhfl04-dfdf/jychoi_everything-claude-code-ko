# Checkpoint 명령어

현재 검증 상태를 저장하고 진행 체크포인트를 생성합니다: $ARGUMENTS

## 작업 내용

다음을 포함한 현재 진행 상태의 스냅샷을 생성합니다:

1. **테스트 상태** - 어떤 테스트가 통과/실패하는지
2. **커버리지** - 현재 커버리지 지표
3. **빌드 상태** - 빌드 성공 또는 오류
4. **코드 변경** - 수정사항 요약
5. **다음 단계** - 남은 작업

## 체크포인트 형식

### 체크포인트: [타임스탬프]

**테스트**
- 전체: X
- 통과: Y
- 실패: Z
- 커버리지: XX%

**빌드**
- 상태: ✅ 통과 / ❌ 실패
- 오류: [있는 경우]

**마지막 체크포인트 이후 변경사항**
```
git diff --stat [last-checkpoint-commit]
```

**완료된 작업**
- [x] 작업 1
- [x] 작업 2
- [ ] 작업 3 (진행 중)

**차단 이슈**
- [이슈 설명]

**다음 단계**
1. 단계 1
2. 단계 2

## 검증 루프와의 연계

체크포인트는 검증 루프와 통합됩니다:

```
/plan → 구현 → /checkpoint → /verify → /checkpoint → 구현 → ...
```

체크포인트 활용:
- 위험한 변경 전 상태 저장
- 단계별 진행 상황 추적
- 필요 시 롤백 지원
- 검증 지점 문서화

---

**팁**: 자연스러운 중단점에서 체크포인트를 생성하세요: 각 단계 완료 후, 대규모 리팩토링 전, 중요한 버그 수정 후.
