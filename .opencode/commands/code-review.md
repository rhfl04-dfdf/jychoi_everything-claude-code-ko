# 코드 리뷰 커맨드

품질, 보안, 유지보수성에 대한 코드 변경 사항 리뷰: $ARGUMENTS

## 작업 내용

1. **변경된 파일 확인**: `git diff --name-only HEAD` 실행
2. **각 파일 분석** - 이슈 점검
3. **구조화된 보고서 생성**
4. **실행 가능한 권장 사항 제공**

## 점검 카테고리

### 보안 이슈 (심각)
- [ ] 하드코딩된 자격증명, API 키, 토큰
- [ ] SQL injection 취약점
- [ ] XSS 취약점
- [ ] 누락된 입력 검증
- [ ] 안전하지 않은 의존성
- [ ] 경로 탐색 위험
- [ ] 인증/인가 결함

### 코드 품질 (높음)
- [ ] 50줄 초과 함수
- [ ] 800줄 초과 파일
- [ ] 4단계 초과 중첩 깊이
- [ ] 누락된 에러 처리
- [ ] console.log 문
- [ ] TODO/FIXME 주석
- [ ] 공개 API에 누락된 JSDoc

### 모범 사례 (중간)
- [ ] 변이 패턴 (대신 불변 사용)
- [ ] 불필요한 복잡성
- [ ] 새 코드에 누락된 테스트
- [ ] 접근성 이슈 (a11y)
- [ ] 성능 우려사항

### 스타일 (낮음)
- [ ] 일관성 없는 명명
- [ ] 누락된 타입 어노테이션
- [ ] 포매팅 이슈

## 보고서 형식

발견된 각 이슈에 대해:

```
**[SEVERITY]** file.ts:123
Issue: [Description]
Fix: [How to fix]
```

## 판정

- **심각 또는 높음 이슈**: 커밋 차단, 수정 필요
- **중간 이슈**: 머지 전 수정 권장
- **낮음 이슈**: 선택적 개선사항

---

**중요**: 보안 취약점이 있는 코드는 절대 승인하지 마세요!
