# Eval 명령어

인수 기준에 대해 구현을 평가합니다: $ARGUMENTS

## 작업 내용

구현이 요구사항을 충족하는지 구조화된 평가를 실행합니다.

## 평가 프레임워크

### 평가 유형

1. **이진 평가** - 통과/실패
   - 작동하는가? 예/아니오
   - 적합한 경우: 기능 완성, 버그 수정

2. **점수 평가** - 0-100점
   - 얼마나 잘 작동하는가?
   - 적합한 경우: 성능, 품질 지표

3. **루브릭 평가** - 카테고리별 점수
   - 여러 차원에서 평가
   - 적합한 경우: 종합 리뷰

## 평가 프로세스

### 단계 1: 기준 정의

```
Acceptance Criteria:
1. [Criterion 1] - [weight]
2. [Criterion 2] - [weight]
3. [Criterion 3] - [weight]
```

### 단계 2: 테스트 실행

각 기준에 대해:
- 관련 테스트 실행
- 증거 수집
- 결과 점수 매기기

### 단계 3: 점수 계산

```
Final Score = Σ (criterion_score × weight) / total_weight
```

## Pass@K 지표

비결정적 평가의 경우:
- K회 실행
- 통과율 계산
- 보고: "Pass@K = X/K"
