# 보안 검토 명령어

포괄적인 보안 검토를 수행합니다: $ARGUMENTS

## 작업 내용

OWASP 지침과 보안 모범 사례에 따라 지정된 코드의 보안 취약점을 분석합니다.

## 보안 체크리스트

### OWASP Top 10

1. **인젝션** (SQL, NoSQL, OS 명령어, LDAP)
   - 매개변수화된 쿼리 확인
   - 입력 살균 검증
   - 동적 쿼리 구성 검토

2. **인증 취약점**
   - 비밀번호 저장 (bcrypt, argon2)
   - 세션 관리
   - 다중 인증
   - 비밀번호 재설정 흐름

3. **민감한 데이터 노출**
   - 저장 및 전송 중 암호화
   - 적절한 키 관리
   - PII 처리

4. **XML 외부 엔티티 (XXE)**
   - DTD 처리 비활성화
   - XML 입력 유효성 검사

5. **접근 제어 취약점**
   - 모든 엔드포인트의 인가 확인
   - 역할 기반 접근 제어
   - 리소스 소유권 검증

6. **보안 설정 오류**
   - 기본 자격 증명 제거
   - 오류 처리가 정보를 누출하지 않는지 확인
   - 보안 헤더 설정

7. **크로스 사이트 스크립팅 (XSS)**
   - 출력 인코딩
   - Content Security Policy
   - 입력 살균

8. **안전하지 않은 역직렬화**
   - 직렬화된 데이터 검증
   - 무결성 검사 구현

9. **알려진 취약점이 있는 컴포넌트 사용**
   - `npm audit` 실행
   - 오래된 의존성 확인

10. **불충분한 로깅 및 모니터링**
    - 보안 이벤트 로깅
    - 로그에 민감한 데이터 없음
    - 알림 설정
