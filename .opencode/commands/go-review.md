# Go 리뷰 커맨드

관용적 패턴과 모범 사례에 대한 Go 코드 리뷰: $ARGUMENTS

## 작업 내용

1. **Go 코드 분석** - 관용구 및 패턴 점검
2. **동시성 확인** - goroutine, channel, mutex
3. **에러 처리 리뷰** - 적절한 에러 래핑
4. **성능 검증** - 할당, 병목 지점

## 리뷰 체크리스트

### 관용적 Go
- [ ] 패키지 명명 (소문자, 밑줄 없음)
- [ ] 변수 명명 (camelCase, 짧게)
- [ ] 인터페이스 명명 (-er로 끝남)
- [ ] 에러 명명 (Err로 시작)

### 에러 처리
- [ ] 에러가 무시되지 않고 확인됨
- [ ] 에러가 컨텍스트와 함께 래핑됨 (`fmt.Errorf("...: %w", err)`)
- [ ] sentinel 에러가 적절히 사용됨
- [ ] 필요 시 커스텀 에러 타입 사용

### 동시성
- [ ] goroutine이 적절히 관리됨
- [ ] channel이 적절히 버퍼링됨
- [ ] 데이터 레이스 없음 (`-race` 플래그 사용)
- [ ] 취소를 위한 context 전달
- [ ] WaitGroup이 올바르게 사용됨

### 성능
- [ ] 불필요한 할당 방지
- [ ] 빈번한 할당에 `sync.Pool` 사용
- [ ] 작은 struct에는 값 리시버 선호
- [ ] I/O 연산 버퍼링

### 코드 구조
- [ ] 작고 집중된 패키지
- [ ] 명확한 의존성 방향
- [ ] 비공개 코드에 internal 패키지 사용
- [ ] export에 Godoc 주석
