# Update Docs 명령어

최근 변경사항을 반영하여 문서를 업데이트합니다: $ARGUMENTS

## 작업 내용

1. **변경된 코드 식별** - `git diff --name-only`
2. **관련 문서 찾기** - README, API 문서, 가이드
3. **문서 업데이트** - 코드와 동기화 유지
4. **정확성 확인** - 문서가 구현과 일치하는지 검증

## 문서 유형

### README.md
- 설치 안내
- 빠른 시작 가이드
- 기능 개요
- 설정 옵션

### API 문서
- 엔드포인트 설명
- 요청/응답 형식
- 인증 상세
- 오류 코드

### 코드 주석
- 공개 API에 대한 JSDoc
- 복잡한 로직 설명
- TODO/FIXME 정리

### 가이드
- 사용법 튜토리얼
- 아키텍처 결정 기록 (ADR)
- 문제 해결 가이드

## 업데이트 체크리스트

- [ ] README가 현재 기능을 반영
- [ ] API 문서가 엔드포인트와 일치
- [ ] 변경된 함수의 JSDoc 업데이트 완료
- [ ] 예제가 정상 작동
- [ ] 링크가 유효
- [ ] 버전 번호 업데이트 완료

## 문서 품질

### 좋은 문서
- 정확하고 최신 상태
- 명확하고 간결
- 작동하는 예제 포함
- 엣지 케이스 설명

### 지양 사항
- 오래된 정보
- 누락된 매개변수
- 깨진 예제
- 모호한 표현

---

**중요**: 문서는 코드 변경과 함께 업데이트해야 하며, 나중에 하는 것이 아닙니다.
